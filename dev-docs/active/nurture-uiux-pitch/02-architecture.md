## 0. 贯穿故事线

**主事件：**"最近宝宝要不到的东西就哭闹"——情绪调节 + 需求表达议题。

选择理由：聚焦孩子内在发展（不是后勤事件）；家/园两侧都会发生，天然需要多方观察；
家庭会真实为它设目标；老师有专业手法可贡献（机构=能力方而非纯数据源）；
以周为单位演进，能展示 目标→观察→反馈→复盘 的短周期闭环
（对应产品模板类 `emotion_behavior_regulation` / `family_rule_trial` 的真实工作流）。

**流转脚本（工作稿，措辞待定）：**

| 步 | 界面 | 发生什么 |
| --- | --- | --- |
| 1 | 家长 Chat | 周日晚家长吐槽"要不到就哭，超市躺地上"。AI 整理近况卡（不判断），问"要不要设为近期关注：用语言表达想要什么，慢慢接受等待？"家长确认。AI 建议"园里也常有类似情境"，家长**显式授权**园所协同观察。 |
| 2 | 老师看板 | 班级工作台出现新关注卡（含授权范围），老师确认接手。 |
| 3 | 老师看板/Chat | 周三沙池：拍照+一句话"想要别人的铲子，没抢，说了'给我玩玩'"。AI 关联宝宝+目标，代拟家长反馈，老师确认发送。 |
| 4 | 家长 Chat | 反向推送卡到达：照片+老师引导方式。家长就地追问 AI"她后来情绪怎么样？"，AI 基于授权数据回答。 |
| 5 | 家长看板 | 当前关注目标进展更新；时间线多源汇入（园所/家来源徽章）。 |
| 6 | 家长看板 | 两周后复盘卡：哭闹频次趋势（观察语言，非评分）、哪些方式有效、下一步微调。 |
| 7 | 机构 mobile | 本周"情绪与表达"类观察增长、家园协同议题进行中、需园长关注的推送。 |
| 8 | 机构 Web | 理念"尊重儿童表达"→阶段目标→活动模板→班级采纳与证据回流；入托邀请流程。 |

日常场景（吃睡/活动/照片）作为环境内容穿插在各界面，避免"全 app 只有一件事"的失真。

## 1. 家长 mobile Chat（AI 互动 + 反向推送）

定位：家长"随口说"的入口 + 园所动态的到达处。**双向流**。

对话流承载三类内容：
1. AI 对话（家长气泡 + AI 结构卡回复：时间/场景/关联/温和建议）；
2. **反向推送卡**（org_to_family）：园所照片、观察反馈、事项回执——卡片形态入流，可展开；
3. 待确认动作卡：设为关注目标 / 授权园所 / 发给老师（AI 代拟、家长可改）。

关键交互：
- **"和 AI 聊"与"发给园所"是两个显式分开的动作**。私域默认不出家庭；任何跨边界一次显式确认。这是全产品最重要的信任设计，demo 必演。
- **对推送内容就地追问**：推送卡下直接问 AI（"她当时情绪怎么样？"），AI 基于已授权数据回答——杀手交互。
- 回执状态链：已送达→老师已确认→已回复。
- 推送分级：日常静默入流（看板聚合），重要才通知——避免通知疲劳。分级规则待共创。
- 所有园所来源内容带溯源（哪位老师/哪个活动）。

文案红线：AI 是整理者不下诊断；健康类一律"如果持续，建议咨询医生"。

### 1a-2. AI 房间与人类房间的区分（2026-07-21 已拍板并落图）

目标：不看头部文字，进错房间 3 秒内能感觉到（安全问题，非纯美观）。四层方案：
1. **形态（主力）**：AI 回复无气泡（左贴边结构块/卡片+行动芯片）、无头像仅星形记号、无已读概念、
   我的消息浅底描边；人类房间经典气泡（头像+姓名+已读回执）、我的消息实心 pine、微信式输入条。
   逻辑：AI 房间="工作台的对话形态"，人类房间="通讯工具"。
2. **氛围**：AI 房间背景极淡色差 + 头部"仅家庭可见"徽章；人类房间标准 ground + 成员行。
3. **入口**：对话 tab 会话列表分组——置顶 AI 助手行（星形、浅色差、无红点、建议预览）+
   "消息"分组（头像+未读角标）。
4. **色彩**：选 A——形态为主色彩克制；AI 房间仅极淡背景色差（#F4F6EF），无专属强调色。
5. 入口（2026-07-21 第五轮修订，已拍板并于 r4 落图）：废弃会话列表——"助手"tab 直达 AI 房间；
   人类沟通（班级群+老师们直聊）全部收进浮标（点开=全屏底部抽屉，segmented 切换两房间，
   角标=未读之和）。心智："找 AI 按 tab，找人点气泡"。底部 tab：看板/助手/授权。
6. 移动端设计基准（已拍板=色板 A，r4 落图）：参考 /Volumes/DataDisk/Project/My-Chat——其 mobile 已有完整
   morethan 设计系统（theme-tokens.ts：cream #F5F2EA 画布/paper 卡片/navy #283E68/orange #E1703C
   primary/暖墨 ink 系，与 web-workbench、portal 同一品牌）。待定：mobile mockups 是否全面
   切换 morethan（推荐，架构诚实：App shell 即 My-Chat），松林绿降级为 Nurture 场景点缀色。
7. 导航 shell（2026-07-21 第六轮讨论，推荐采用）：弃底部 tab，改 My-Chat 左抽屉 shell
   （实证：NavDrawer = UpperRail surface 切换 + LowerContextHost 下半段上下文 + footer 设置）。
   - 家长抽屉：rail 看板/助手；中段"孩子"分组（多孩切换——此前未解决的真实需求）+
     per-surface 上下文（助手→最近话题；看板→三视图快捷）；底部 授权（状态行）/设置。
   - 老师抽屉：rail 班级流/收件箱(角标)/助手；中段"我的班级"；底部 设置。
   - 高频动线不经抽屉（开屏直达看板 + 浮标找人）；顶部栏汉堡+位置标题；浮标成唯一悬浮元素。
   - 代价与化解：授权从 tab 降为抽屉项（带状态行+园区视图保留入口）。
8. **场景视图机制（2026-07-21 第七轮，用户提出）**：My-Chat 主界面增加视图切换——
   **宝宝视图（Nurture 场景接管 shell）/ 通用视图（My-Chat 原生）**，切换器在抽屉顶部身份区。
   视图控制：默认落点（宝宝→看板；通用→AI 聊天窗）、rail 组成（看板·助手 vs 聊天·论坛·看板）、
   抽屉下半段上下文、悬浮元素（宝宝视图才有班级群浮标）、AI 默认上下文（携带孩子 vs 通用）。
   原则：视图只改默认入口与上下文，不改权限与数据边界。老师"班级视图"、园长"园所视图"
   为同一 lens 机制的角色实例；多孩在宝宝视图内部切换（视图=场景镜头，非对象实例）。
   Demo 处理：十屏均为宝宝视图；通用视图不单独画屏，仅在抽屉 mockup 顶部露出切换器。
附带效果：老师 Chat 的工作卡片更自然；AI 房间内"待发送卡"因形似人类气泡而天然醒目（边界加固）。

### 1b. 家园直聊（2026-07-21 新增）

**三个对话空间**：① 家庭私域（家长+AI，默认不出家庭）；② **家园直聊**（该家庭家长们+班级老师们，
家↔园正式通道）；③ 班级群（全班公共空间）。"老师不进家庭私聊"原则指①，②是独立通道。

- 形态：每个家庭一个小群（妈妈爸爸+李老师王老师），**用法与微信无区别**；
- 三个静默升级：已读回执；重要信息自动挂到宝宝记录（带"已记入·今日叮嘱"提示芯片）；
  发送动作本身即显式跨边界确认（无额外弹窗）；
- 入口：家长端"对话"tab 升级为**会话列表**（AI 助手/老师们/班级群，My-Chat shell 自然形态），
  浮标仍是班级群快捷方式；老师端从**收件箱**进入各家庭会话（收件箱=家庭会话列表+待办）。

## 2. 家长 mobile 看板

**核心纠偏（2026-07-21 对齐）：看板不是机构导向，而是以家庭共同选定的培养理念 +
长短期目标为核心的成长档案。机构只是数据源与聚合方式之一。**
（正对应家庭生态的 Family Charter / Current Focus / Development Vector 概念。）

**三视图切换（2026-07-21 更新）**：顶部切换 **养育 / 成长线 / 园区**。
- 养育视图 = 家庭成长档案快照（默认，"现在时"，不参与时间翻页）；
- 成长线视图 = 翻相册式纵向无限流：大图卡+紧凑文字行混排、点击展开（完整观察+关联目标+回家怎么接）、
  月份/入托周数分隔锚、里程碑"第一次"事件星标、筛选芯片（全部/照片/里程碑/仅园所/仅家庭）、支持日期跳转；
- 园区视图 = 机构导向聚合，**一天一页的"日记本"**：左右横滑翻日（电子书式）+ 日期头点击弹日历
  （有记录日带密度点、"回到今天"钮）；空白日也有页（"周末在家·N 条家庭记录"/"未入园"）。
  行政类内容（通知/食谱/缴费）全部住这里。
- 交互隐喻：**横向翻"日子"，纵向翻"成长"**。
- **班级群 = 对话浮标**：看板三视图内浮动气泡（多人图标+"班级群"微标签+未读角标，与 AI 对话入口视觉区分），
  点击弹**全屏底部抽屉**展示群聊式班级流（老师头像+名字+时间+图文；老师发一次、各家长仅见授权范围）。
  互动尺度（2026-07-21 确认）：家长可自由发言，**使用上与微信群无区别**；AI 仅做温和分流
  （识别涉及个体宝宝的问题时提示私聊跟进），不设发言限制。

**养育视图三层目标结构（2026-07-21 定）**：养育理念 → 长期目标 → 短期目标，
呈"目标阶梯"折叠展示；**主事件归属短期目标**（"能用语言表达想要什么，慢慢接受'等一下'"），
卡片上向上关联长期目标与理念（如 理念"尊重表达" › 长期"学会与情绪相处" › 短期·进行中）。

**养育视图定位（2026-07-21 更新）**：养育体系是"移动 + 网页"双端共同完成的——
移动端只是轻看板（显示当前关注重点 + 常用操作），具体数据与配置在**家庭网页工作台**
（即 Pilot-0-B 锁定的 Guardian family workbench，不在六界面 demo 范围，界面留跳转提示行即可）。

养育视图模块顺序（自上而下，简化版）：
1. **今日一瞥**：一句话日结（带日期）；
2. **当前关注**：1-3 张关注点卡（主事件卡高亮：起始日期/周次 + 单线周趋势微图 + 多源观察计数；
   次要关注点轻量卡展示多关注点能力）；
3. **方向行**（紧凑单行）：理念 › 长期目标，点击去网页端看详情；
4. **常用操作**：＋新建关注点 / 记一笔（家庭观察）；
5. 底部提示行：完整数据与设置 · 家庭网页工作台。
（原"目标阶梯"三层卡收敛为 关注卡+方向行；成长时间线移至成长线视图。）

常驻入口：授权管理（园所能看什么、随时撤）——不占篇幅但必须常在。

对园长的讲述框架注意：机构定位是"共同养育者/最重要的协同方"，
时间线中园所来源内容占比高、质量高 → 园所价值凸显，但数据主权在家庭。措辞避免让园长感到被降格为"供应商"。

## 3. 老师 mobile Chat

**架构对齐（2026-07-21）：与家长 Chat 是同一套 My-Chat 界面。**
AI 按角色+场景上下文自动判断内容处理方式（Nurture 场景 handler），给出合适的后续交互引导。

- 同一视觉 shell；差异只在顶部角色/上下文徽标与 AI 行为。
- 老师输入偏"记录/查询"：语音一句话 → AI 结构化（识别孩子、归类、关联目标）→
  回复附**动作芯片**："记入宝宝观察 / 发家长版反馈 / 加入今日待办"——可改选，确认后生效。
- AI 判断错的兜底：芯片永远可改，跨边界动作永远显式确认（UX 即治理）。
- demo 叙事点：**"一套对话，各自的助手"**——两页同 shell 对照本身就是一页故事。

## 4. 老师 mobile 看板（班级工作台）

**定位纠偏（2026-07-21）：老师最常用的是看板而非 Chat——减负主战场在这里。**

设计原则：类"微信群"交互过渡，不影响老师正常使用习惯。

- 主体是**班级流**：时间序图文卡 + 轻量输入框，视觉像群聊/朋友圈混合体，零学习成本。
- 与微信群的三个静默升级（demo 讲解点）：
  1. 每条内容结构化归属（发布时点头像圈孩子）；
  2. 自动分发（每个家长只看到自家+授权范围内的）；
  3. 可检索沉淀（不再是聊天记录坟场，自动归档到孩子/活动/目标）。
- **拍照第一公民**：底部中央大拍照键（微信肌肉记忆）。流程：拍照 → 点头像网格圈孩子 →
  AI 建议分类/关联 → 发布，三步内完成。
- 顶部"今日班级"一览条：每娃状态点（家长叮嘱/待观察/正常）。
- 收件箱（2026-07-21 确认）："置顶待办条 + 独立 tab"双入口，家长私信原文不混入班级流。
- **卡片解剖（2026-07-21 定，分发确认卡与流内卡共用）**：
  头像区（1-N 宝宝头像芯片，多孩合照多头像，"+"可编辑——发布前改零成本；发布后改=触发撤回+回执，
  底层即 revoke/redaction 语义，demo 只演发布前）
  / 内容区（图片、视频缩略图+时长、文字；音频→转写文字为主体+播放芯片留原音）
  / 元数据行（时间·场景归类·命中目标）
  / 状态区（草稿态：一键确认发布（默认全去向）；已发布态：去向回执芯片"家庭×N ✓·班级相册 ✓·目标记录 ✓"）。
- **发布范式（2026-07-21 第四轮定稿方向：倒计时自动发布）**：
  **拍完/说完即结束；AI 聚簇整理成一或多张卡片顺序入流，各带倒计时读条，
  到点自动发布；发布后卡片收缩成一行状态提示，可查看/修改（撤回替换+回执）。**
  - 卡片生命周期四态：整理中（无读条）→ 待发布（读条）→ 已发布（状态行）→ 查看/修改；
  - AI 聚簇决定卡片粒度（12 张照片→1 张活动卡；观察→单独卡）；顶部总控"全部暂停/立即全部发布"；
  - "关联宝宝"三源：照片识别 / 文字语音点名 / 活动上下文；
  - **两段式发布模型（2026-07-23 第九轮拍板，替代下方四道护栏的治理论证）**：
    读条走完 ≠ 发给家长，而是**进入待发队列**——内容以"未发送"态停在目标会话里
    （老师可见的灰虚线气泡，标"待发送·今天 17:00"+现在发送/撤回）；真正跨家庭边界发生在发送时刻：
    手动发=显式确认；定时批量发（默认 12:30/17:00，机构可配）=显式配置的常设策略。
    到点默认自动发出 + 离园前提醒（"今天有 N 条将于 17:00 发出"）；低置信/敏感永远手动；
    园所可整体切"全手动"。**只有 AI 整理内容走队列**——老师亲手打的字/回复永远即时。
    价值：拆开"内容对不对"（读条否决窗）与"何时打扰家长"（发送时机）两个问题；
    家长从零散推送变成午睡/离园两包"当日小结"；治理上自动化部分从未触碰家庭边界。
  - 原四道护栏（历史，第 1/2/4 条仍有效，第 3 条"发布后可撤"变为队列内撤回+发出后撤回两级）：
    1. 置信门控：仅高置信内容走读条自动发；待关联/识别模糊/敏感类（情绪低落、磕碰、健康）永不自动发；
       待关联内容只进班级档案、进不了家庭时间线；
    2. 触碰即暂停：碰卡片读条停，进入编辑；
    3. 发布后可撤：状态行→修改=撤回替换+回执；
    4. 机构可配：读条时长可调、自动发布可整体关闭（园长共创旋钮，demo 明说）；
  - 主场归属（2026-07-21 确认）：**班级流**；AI 对话仅镜像/控制；

## 7. 横切 UX 原则（六界面共用）

1. 同一事实，三种投影（同一条记录在家长/老师/机构端不同形态，demo 用主事件演）；
2. 跨家园边界必显式（确认动作 + 回执状态）；
3. AI 只做整理者（产出永远可编辑、需确认，不下判断）；
4. 状态可见（发出→已读→已回，信任靠回执建立）;
5. 只有观察与趋势，没有分数与排名（数字一律"校准与支持"框架）。
